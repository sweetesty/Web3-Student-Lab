# Web3 Glossary

This glossary explains common Web3 terms used in this lab. It is designed for students who are new
to blockchain and decentralized applications.

---

## DAO (Decentralized Autonomous Organization)

A DAO is a group or organization that is run by code and community voting instead of a central
authority.

- Decisions are made by members through voting
- Rules are written in smart contracts
- No single person controls everything

Think of it like an online group project where everyone votes on decisions instead of having a boss.

---

## Gas

Gas is the fee you pay to perform actions on a blockchain.

- Used when sending transactions or running smart contracts
- Paid to network validators/miners
- Prevents spam and misuse of the network

Similar to paying a small fee to use a service.

---

## WASM (WebAssembly)

WASM is a fast, low-level programming format that allows code to run efficiently on different
platforms.

- Used to run smart contracts in some blockchains
- Faster and more secure than traditional scripting
- Language-independent (can compile from Rust, C, etc.)

Think of it as a universal format that helps programs run quickly everywhere.

---

## Smart Contract

A smart contract is a program stored on the blockchain that runs automatically when certain
conditions are met.

- No middleman required
- Cannot be changed easily once deployed
- Used for payments, agreements, and automation

Like a digital agreement that executes itself.

---

## XLM (Stellar Lumens)

XLM is the native cryptocurrency of the Stellar network.

- Used to pay transaction fees
- Used for sending and receiving money
- Helps move value across borders quickly

Think of it as the “currency” used inside the Stellar system.

---

## Soroban

Soroban is the smart contract platform built on the Stellar network.

- Allows developers to build decentralized apps (dApps)
- Uses WASM for running contracts
- Designed to be fast, scalable, and developer-friendly

It’s the tool that lets you build apps on Stellar, similar to how Ethereum uses Solidity.

---

## Blockchain

A blockchain is a distributed digital ledger that records transactions securely.

- Data is stored in blocks
- Blocks are linked together in a chain
- Cannot be easily changed or hacked

Like a shared notebook that everyone can see, but no one can secretly edit.

---

## Wallet

A wallet is a tool used to store and manage your digital assets.

- Stores your private keys
- Allows you to send and receive crypto
- Can be mobile, web, or hardware-based

Like a digital bank account for crypto.

---

## Private Key

A private key is a secret code that gives you access to your crypto.

- Must never be shared
- Used to sign transactions
- Whoever has it controls the funds

Like your ATM PIN — keep it safe.

---

## Public Address

A public address is the identifier you share to receive funds.

- Safe to share
- Generated from your wallet
- Used like an account number

Like your bank account number.

---

## Transaction

A transaction is any action recorded on the blockchain.

- Sending or receiving crypto
- Interacting with smart contracts
- Permanent once confirmed

Like sending money or making a payment online.

---

## dApp (Decentralized Application)

A dApp is an application that runs on a blockchain instead of a central server.

- Uses smart contracts
- More transparent and secure
- No single point of control

Like a normal app, but not controlled by one company.

---

## Final Note

Web3 can feel confusing at first, but these terms form the foundation. As you build projects and
interact with blockchain systems, these concepts will become clearer over time.
